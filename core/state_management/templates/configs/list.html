建立分析設定提示

  • 先決定 analysis_method_id，需與分析服務訂閱的 routing key 一致。
  • config_id 由系統產生，路由規則會在 actions 中引用它。
  • parameters 必須是有效 JSON，可包含模型路徑、特徵參數與後處理選項。
  • 停用的設定不會被調度器引用，請確認與路由規則一致後再切換狀態。

詳細說明見 docs/設定規則說明.html 的「分析設定」。
